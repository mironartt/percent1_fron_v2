Есть страница: `/land/newyear`

Некоторые доработки по ней:
- если пользователь авторизован то:
    - в шапке не должно быть кнопки "войти" - вмето должно быть "личный кабинет" и вести на главную личного кабинета (`app`) (аналогично и для главной страницы сайта `/` - если пользователь авторизован, не должно быть кнопок "войти" и "начать бесплатно", а должна быть одна - "личный кабинет" (или как-то получше придумать назавние для этой кнопки))
- тест на главной (кнопка "Начать тест →") `/land/newyear/test` - это посути рефлексия по ССП.  
Поэтому получается аналогия: 
    - оценка категории: это оценка колеса ССП
    - ответ на последующий вопрос это ответ в рефлексии в поле "Почему такой балл?" (поле `rating_reason`)
Это значит что тебе нужно писать в локал-сторадж данные указанные пользователем во время заполнения текста.

По завершению теста пользователем отправить запрос об этом на бэк со всеми нужными данными (эндпоинты и документация по ингерации с бэком будет приложена).  
На бэке данному результату теста будет присвоен id и hash. Тебе нужно также запомнить эти данные.  
Если во время прохождения теста, пользователь не был авторизован, то после его авторизации/регистрации тебе нужно будет отправить эндпоинт о том, чтобы связать запись (по сохраненному раннее hash) с данным пользователем.  

Возвращенный бэком hash, также будет являться частью сылки на страницу результатов теста.  
Т.е. линк результата долджен быть формата `/land/newyear/results/<hash>`.  
И данный линк будет публичным.  
Т.е. если какой-нибудь пользователь откроет данный линк, то фронт должен взять указанный hash и отправить на бэк запрос за получением данных о результатах и отрисовать его. 
Пока по данным из бэка будут отрисовываться только данные для верхнего блока "Твои итоги 2025. Анализ твоего года по 6 ключевым сферам жизни".  
Остальные блоки:
- "Главный рычаг 2026"
- "Цели на 2026"
- "План на январь"
Фронт пока должен рисовать сам.  


Новые эндпоинты:
1. POST /api/rest/front/app/landing-ssp-test/save/ - публичный, сохранение теста
2. POST /api/rest/front/app/landing-ssp-test/get/ - публичный, получение по хэшу
3. POST /api/rest/front/app/landing-ssp-test/link/ - приватный, привязка к пользователю

Анонимный пользователь может создавать результаты не чаще раза в минуту.

Детали по эндпоинтам в прикрепленном файле `landing_ssp_test_api.md`
