Для фронтенда: Проверка лимитов (бэкенд)

Что изменилось  

Бэкенд теперь проверяет лимиты при создании целей и привычек и возвращает информативные ошибки.

Новые ошибки

При создании цели сверх лимита (POST `/api/rest/front/app/goals/update/`):
```json
{
  "error": "goals_limit_exceeded",
  "message": "Достигнут лимит целей для вашего тарифа (4)...",
  "current_count": 4,
  "limit": 4,
  "tariff_code": "free"
}  
```

При создании привычки сверх лимита (POST `/api/rest/front/app/habits/update/`):
```json
{
  "error": "habits_limit_exceeded",
  "message": "Достигнут лимит привычек для вашего тарифа (3)...",
  "current_count": 3,
  "limit": 3,
  "tariff_code": "free"
}  
```
 
При создании достижений сверх лимита (POST `/api/rest/front/app/habits/rewards/create/ `):
```json
{
  "error": "rewards_limit_exceeded",
  "message": "Достигнут лимит наград для вашего тарифа (5). Перейдите на Pro для безлимитных наград.",
  "current_count": 5,
  "limit": 5,
  "tariff_code": "free"
} 
```
                                                                                              
                                                                                                                                     
Изменения в лимитах:                                                        
- Убран max_diary_entries_per_month - дневник теперь без лимитов
- Добавлен max_rewards - лимит достижений (Free: 5, Pro/Club: безлимит)
                                                                             
 
Что делать:
1. Обрабатывать ошибки goals_limit_exceeded и habits_limit_exceeded (HTTP 400)   
2. Показывать модалку upgrade с информацией из ответа
3. Достижения не проверяются — они разблокируются автоматически
4. Дневник/планирование/ССП не проверяются — всегда доступно
 
Лимиты 

| Тариф   | Цели     | Привычки | Награды  | Дневник     |                                                                         
|---------|----------|----------|----------|-------------|                                                                         
| Free    | 4        | 3        | 5        | Без лимитов |                                                                         
| Pro     | Безлимит | Безлимит | Безлимит | Без лимитов |                                                                         
| Клуб 1% | Безлимит | Безлимит | Безлимит | Без лимитов |  


Т.е. фронт изначально должен сам смотреть может ли пользователь создать ту или иную сущность и если кончились лимит, то отобразить модалку о покупке/продление тарифа, но если как-то так получилось, что запрос ушел, и фронт видит в ответ указанные ошибки то он должен отобразить модалку о покупке/продление тарифа
