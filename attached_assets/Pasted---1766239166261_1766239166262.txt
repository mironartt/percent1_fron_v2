    Кейс: пользователь находится на странице "детали по цели", где он создает шаги для цели. После создания всех нужных шагов, на данной странице у пользователя есть кнопка "запланировать цель".

Привет!

Добавил новый функционал в эндпоинт POST /api/rest/front/app/goals/get/

**Что добавлено:**
Новый опциональный параметр first_goal_id (integer), который позволяет указать ID цели для приоритетного отображения первой в списке.

**Кейс использования:**
Когда пользователь на странице "детали по цели" создает шаги и нажимает "запланировать цель",
при редиректе на страницу планирования эта цель должна быть первой в списке.

**Как использовать:**
POST /api/rest/front/app/goals/get/
{
"first_goal_id": 123,  // ID цели, которая должна быть первой
"order_by": "date_created",
"order_direction": "desc",
// ... остальные параметры как обычно
}

**Важно:**
- Параметр полностью опциональный - если не передан, все работает как раньше
- Если передан несуществующий ID или чужая цель, вернется ошибка goal_not_access
- Цель будет первой, даже если по сортировке она должна быть в другом месте
- Фильтры применяются как обычно - если first_goal_id не проходит фильтры, она не попадет в результаты
