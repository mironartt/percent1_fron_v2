Я поставил для бэкенда задачу:
```
Нужна разработка/доработка эндпоинтов связанных с разделом "достижения".  

### Общая информация:
Данный раздел предоставляет пользователю возможность награждать себя определенными объектами достижений списывая заработанные XP на них.  
Т.е. пользователь зарабатывает XP выполняя различные события (привычки, записи в дневник, выполнение запланированных шагов и тд...), и 
после он может потратить их на "достижения" которые он добавил ранее.  

Термин "достижения" по сути является наградами пользователя для себя.  
Т.е. пользователь сам создает для себя "достижения" (награды), назначает им цену (какое кол-во XP стоит данная награда) и копит на них выполняя задания поставленные себе.  

----

### Реализация в интерфейсе пользователя:
- Снимок экрана1 - главная страница "достижения".   
Данная страница содержит блоки:
    - "верхний бар" - отображает 3 значения связанные с заработанными XP:
        - XP за сегодня
        - XP за неделю
        - XP за все время  
    Также в нем содержится информация о текущем балансе XP у пользователя.
    - "центральный блок" - список всех доступных "достижений" добавленных пользователем (отображается в порядке сортировки: ближе всего к доступности покупки, а второй ключ сортировки: меньшая стоимость)  
    Для данного блока на бэке должна быть возможность поиска по названиям "достижений", и возможность сортировок по: размер XP, осталось XP до достижения цели (на фронте еще не реализовано)
    - "нижний блок истории списаний/зачислений" - здесь отображается история всех зачислений XP и их списаний. Т.к. записей здесь может быть много, то нужно будет объединять их в общие блоки.  
    К примеру зачисления за выполнения привычек за один день, объединять в одну запись, в которой будет добавлена информация об общей сумме зачисления, дате, из каких частей (весь список за какие привычки было зачисление).
    У привычек могут быть доступны штрафы, поэтому их нужно аналогично отображать.  
    Т.е. в общем для блока истории будут поля:
        - статус - зачисление/списание
        - размер XP
        - тип записи:
            - награда за выполнение привычек за день
            - штраф за НЕ выполнение привычек за день
            - награда за заполнение дневника
            - штраф за НЕ заполнение дневника
            - награда за выполнение планов на день
            - штраф за НЕ выполнение планов на день
            - траты на "достижения"
            - остальное
        - дата
        - данные по связанным сущностям
        - остальные необходимые поля
    Для данного блока должна быть возможность поиска по тексту, и фильтрации по типу списания (за дневник, за привычки, за покупку достижения и тд...), и фильтрация по типу операции: списание/зачисление.  
    Также должна быть пагинация по 10 элементов.

- Снимок экрана2 - модальное окно добавление "достижения".  
В нем пользователь может указать:
    - иконку 
    - название
    - описание
    - задать стоимость в XP для данного "достижения"
- Снимок экрана3 - модальное окно редактирования "достижения".  
Пользователь может редактировать все поля "достижения".  
Также можно удалить "достижение" (на бэке не должно происходить фактического удаления, а лишь заполняться дата удаления, и больше не данное достижение не будет отображаться пользователю).
```
