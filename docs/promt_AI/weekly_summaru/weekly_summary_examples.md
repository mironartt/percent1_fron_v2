# ПРИМЕРЫ НЕДЕЛЬНЫХ ИТОГОВ ПО СЦЕНАРИЯМ

## СЦЕНАРИЙ 1: Сильная неделя (9 предложений)

**Входные данные:**
```json
{
  "user_name": "Дима",
  "week_summary": {
    "tasks_completed": 42,
    "tasks_planned": 50,
    "xp_earned": 320,
    "habits": [
      {"title": "Утренняя зарядка", "completed_days": 7, "total_days": 7},
      {"title": "Медитация 10 минут", "completed_days": 5, "total_days": 7}
    ],
    "best_day": {"day": "Вторник", "completed": 6, "planned": 6}
  },
  "goals_progress": [
    {
      "title": "Запустить MVP",
      "progress_description": "Больше половины работы готово",
      "key_step_completed": "Интеграция API"
    }
  ],
  "near_reward": {
    "reward_name": "Новые наушники",
    "remaining_xp": 180
  },
  "context_insights": {
    "pattern_already_discussed": false
  }
}
```

**Правильное уведомление:**
```
Дима, неделя завершена.

Закрыл 42 задачи из 50 — сильная неделя. Привычки: зарядка всю неделю без пропусков, медитация 5 дней из 7. Заработал 320 XP.

Лучший день — вторник (6 из 6), там была «Интеграция API». Это двинуло MVP — теперь больше половины работы готово.

На следующей неделе продолжай в том же духе. Ещё 180 XP — и «Новые наушники» твои.
```

## АНТИПАТТЕРНЫ (НЕПРАВИЛЬНЫЕ ПРИМЕРЫ)

### ❌ ПЛОХО: Используются проценты

```
Дима, неделя завершена.

Закрыл 35 задач из 50 — 70%. Привычки: зарядка 86%, медитация 57%. Заработал 280 XP.
```
**Почему плохо:** Проценты запрещены, нужны словесные описания

### ❌ ПЛОХО: Повторяется вопрос, который уже задавался

```
Дима, неделя завершена.

Закрыл 28 задач из 50. Зарядка 6 из 7, медитация 4 из 7. Заработал 240 XP.

Четверг-пятница провал. В чём дело — усталость накопилась?
```
**Почему плохо:** Если в четверг/пятницу этот вопрос уже задавался и пользователь ответил, повторять нельзя

### ❌ ПЛОХО: Нет позитивного акцента в слабую неделю

```
Дима, неделя завершена.

Закрыл 18 задач из 45 — меньше половины. Зарядка 3 из 7, медитация 1 раз. Заработал 120 XP.

Неделя провальная. Привычки не делаешь, задачи не закрываешь. Надо взяться за себя.
```
**Почему плохо:** Осуждающий тон, нет позитива, токсичная мотивация

### ❌ ПЛОХО: Перечисление всех проблем сразу

```
Дима, неделя завершена.

Закрыл 30 задач из 50. Заработал 250 XP.

Проблемы: четверг-пятница спад, привычки провисли, MVP стоит месяц, здоровье на 4, а карьера на 8, в дневнике про усталость пишешь. Надо что-то менять.
```
**Почему плохо:** Выбрать нужно ОДИН главный инсайт, не сыпать всё сразу

### ❌ ПЛОХО: Неконкретная рекомендация

```
Дима, неделя завершена.

Закрыл 28 задач из 50. Зарядка 6 из 7. Заработал 240 XP.

Понедельник-среда стабильно, четверг-пятница провал.

На следующей неделе постарайся лучше. Всё получится!
```
**Почему плохо:** "Постарайся лучше" — не actionable рекомендация, клише

### ❌ ПЛОХО: Нет главного инсайта

```
Дима, неделя завершена.

Закрыл 35 задач из 50 — больше половины. Зарядка 6 из 7, медитация 4 из 7. Заработал 280 XP.

Лучший день — вторник (6 из 6).

Продолжай в том же духе!
```
**Почему плохо:** Нет анализа, нет инсайта, просто пересказ цифр

## КЛЮЧЕВЫЕ ПРИНЦИПЫ

✅ **Обязательные элементы:**
- Приветствие: "{{user_name}}, неделя завершена."
- Цифры БЕЗ процентов (только слова)
- Один главный инсайт
- Позитив даже в провал
- Конкретная actionable рекомендация
- Проверка повторения вопросов!

**Формулировки без процентов:**
- Больше/меньше половины, треть, четверть, почти все
- Сильная/средняя/слабая/неровная неделя
- Всю неделю/через раз/только в начале
- Стабильно/провал/спад/рост

❌ **Избегай:**
- Процентов (70%, 86%)
- Повторения вопросов из недели
- Осуждения
- Клише ("постарайся", "всё получится")
- Перечисления всех проблем
- Отсутствия позитива
