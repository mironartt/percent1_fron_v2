# ПРИМЕРЫ: Relationship Checkin

---

## FIRST CHECKIN (~14 дней)

### Пример 1.1: Стандартный первый checkin

**Входные данные:**
```json
{
  "user_name": "Дмитрий",
  "days_in_app": 14,
  "checkin": {
    "type": "first"
  }
}
```

**Ожидаемый результат:**
```
Дмитрий, две недели вместе. Как тебе формат общения? Что-то помогает, что-то может раздражать — нормально об этом сказать.

[Всё ок] [Хочу по-другому]
```

---

### Пример 1.2: Первый checkin с высоким engagement

**Входные данные:**
```json
{
  "user_name": "Мария",
  "days_in_app": 15,
  "checkin": {
    "type": "first"
  },
  "interaction_stats": {
    "messages_last_2_weeks": 24,
    "response_rate": 80
  }
}
```

**Ожидаемый результат:**
```
Мария, две недели работаем. Вижу, активно общаемся — это хорошо. Но вопрос: формат подходит? Или что-то хочется изменить?

[Подходит] [Есть идеи]
```

## TRIGGERED CHECKIN (по сигналу)

### Пример 3.1: Notification fatigue

**Входные данные:**
```json
{
  "user_name": "Павел",
  "days_in_app": 45,
  "checkin": {
    "type": "triggered",
    "trigger_reason": "notification_fatigue"
  },
  "interaction_stats": {
    "notification_open_rate": 25
  }
}
```

**Ожидаемый результат:**
```
Павел, вижу — уведомления открываешь реже. Может, их слишком много? Или формат не тот? Скажи — подстроимся.

[Меньше уведомлений] [Другой формат] [Всё ок]
```

## ПОСЛЕ ПРЕДЫДУЩЕГО CHECKIN

### Пример 4.1: Предыдущий был негативным

**Входные данные:**
```json
{
  "user_name": "Андрей",
  "days_in_app": 45,
  "checkin": {
    "type": "regular"
  },
  "previous_checkin": {
    "date": "2 недели назад",
    "response": "Хочу меньше сообщений",
    "changes_made": "Снижена частота уведомлений"
  }
}
```

**Ожидаемый результат:**
```
Андрей, две недели назад говорил про частоту сообщений. Снизили. Как сейчас — лучше?

[Да, лучше] [Ещё меньше] [Можно вернуть]
```

## АНТИПРИМЕРЫ (как НЕ надо)

### Плохо: Самобичевание
```
Привет! Я хочу стать лучше для тебя. Прости, если что-то было не так. Пожалуйста, скажи, как я могу улучшиться?
```

### Плохо: Формальный опрос
```
Пожалуйста, оцените качество работы AI-наставника по шкале от 1 до 10.
```

### Плохо: Слишком много вопросов
```
Как тебе формат? А частота? А тон? А длина сообщений? А время отправки? Что изменить?
```

### Плохо: Давление на ответ
```
Очень важно, чтобы ты ответил на этот вопрос. Мне нужна твоя обратная связь прямо сейчас.
```

### Плохо: Игнорирование контекста
```
(При low engagement)
Привет! Как тебе формат? Всё отлично работает?
```

---

## ОБРАБОТКА ОТВЕТОВ

### Если "Всё ок" / "Норм":
```
Хорошо. Если что — скажи.
```

### Если "Хочу изменить" / "Есть идеи":
```
Слушаю. Что именно?
```

### Если "Меньше сообщений":
```
Понял. Снижаем частоту. Буду писать только когда реально важно.
```

### Если "Нужна пауза":
```
Окей. Напишу через неделю — или напиши сам, когда будешь готов.
```

### Если негативный feedback:
```
Услышал. Спасибо, что сказал. Адаптируемся.
```

## ЧЕКЛИСТ КАЧЕСТВА

Перед отправкой проверь:

- [ ] 2-4 предложения
- [ ] Один вопрос
- [ ] Нет "прости если что"
- [ ] Нет формальных оценок (1-10)
- [ ] Учтён контекст (trigger_reason если есть)
- [ ] Варианты ответа релевантны ситуации
- [ ] Тон партнёрский, не просящий
