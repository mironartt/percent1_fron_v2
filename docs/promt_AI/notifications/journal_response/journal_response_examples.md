# ПРИМЕРЫ: Journal Response

---

## ПОЗИТИВНАЯ ЗАПИСЬ (хороший день)

### Пример 1.1: Конкретные достижения

**Входные данные:**
```json
{
  "user_name": "Дмитрий",
  "journal_entry": {
    "done": "Закрыл 5 задач по проекту. Наконец-то сдал отчёт, который висел неделю. Позвонил клиенту — договорились о встрече.",
    "not_done": "Не успел позаниматься английским",
    "learned": "Когда начинаю с неприятного — день идёт легче",
    "tomorrow": "Подготовить презентацию к встрече"
  }
}
```

**Ожидаемый результат:**
```
Отчёт, который висел неделю — закрыт. Это не мелочь. Ты пишешь, что начал с неприятного и день пошёл легче. Это работает не у всех — как ты себя заставил начать именно с него?
```

---

### Пример 1.2: Эмоциональный позитив

**Входные данные:**
```json
{
  "user_name": "Мария",
  "journal_entry": {
    "done": "Пробежала 5 км! Первый раз без остановки. Ноги болят, но я довольна.",
    "not_done": "",
    "learned": "Главное — не думать о финише, а фокусироваться на следующем шаге",
    "tomorrow": "День отдыха, но сделаю растяжку"
  }
}
```

**Ожидаемый результат:**
```
5 км без остановки — это реальный прорыв. "Фокус на следующем шаге, не на финише" — это работает не только в беге. В каких ещё задачах можно так же?
```

## НЕГАТИВНАЯ ЗАПИСЬ (сложный день)

### Пример 2.1: Провал задач

**Входные данные:**
```json
{
  "user_name": "Алексей",
  "journal_entry": {
    "done": "Почти ничего",
    "not_done": "Всё. Весь день прокрастинировал. Смотрел видео вместо работы.",
    "learned": "Что у меня нет силы воли",
    "tomorrow": "Попробовать снова"
  }
}
```

**Ожидаемый результат:**
```
Тяжёлый день. "Нет силы воли" — вряд ли правда, скорее что-то другое. Что происходило перед тем, как начал смотреть видео? Может, дело не в воле, а в чём-то конкретном.
```

---

### Пример 2.2: Внешние обстоятельства

**Входные данные:**
```json
{
  "user_name": "Ольга",
  "journal_entry": {
    "done": "Только рутину — готовка, уборка",
    "not_done": "Работа над проектом. Ребёнок болел весь день, не было ни минуты.",
    "learned": "",
    "tomorrow": "Если ребёнку лучше — вернуться к проекту"
  }
}
```

**Ожидаемый результат:**
```
Когда ребёнок болеет — всё остальное отходит. Это нормально, не провал. Рутина тоже была сделана. Как сейчас — есть силы или нужен отдых?
```

## ИНСАЙТ

### Пример 4.1: Осознание паттерна

**Входные данные:**
```json
{
  "user_name": "Виктория",
  "journal_entry": {
    "done": "Сделала презентацию за 2 часа. Обычно трачу день.",
    "not_done": "",
    "learned": "Когда ставлю таймер и говорю 'сколько есть — столько есть', делаю быстрее и не хуже. Перфекционизм только мешает.",
    "tomorrow": "Попробовать так же с отчётом"
  }
}
```

**Ожидаемый результат:**
```
"Сколько есть — столько есть" и перфекционизм отпустил. Это мощный инсайт. Презентация за 2 часа вместо дня — и качество не упало. Что ещё съедает время из-за перфекционизма?
```

---

### Пример 4.2: Осознание про себя

**Входные данные:**
```json
{
  "user_name": "Павел",
  "journal_entry": {
    "done": "Работа, рутина",
    "not_done": "Не позвонил маме, хотя хотел",
    "learned": "Я откладываю не потому что некогда, а потому что не знаю о чём говорить. Это странно.",
    "tomorrow": "Позвонить. Просто спросить как дела."
  }
}
```

**Ожидаемый результат:**
```
"Не знаю о чём говорить" — честное наблюдение. Не про время, а про что-то глубже. Когда последний раз был разговор, который шёл легко?
```

## BURNOUT-СИГНАЛЫ

### Пример 6.1: Усталость

**Входные данные:**
```json
{
  "user_name": "Елена",
  "journal_entry": {
    "done": "Еле доползла до вечера. Сделала минимум.",
    "not_done": "Всё остальное. Нет сил ни на что.",
    "learned": "Что я устала. Очень.",
    "tomorrow": "Не знаю. Наверное то же самое."
  }
}
```

**Ожидаемый результат:**
```
"Устала. Очень." — это не жалоба, это факт. Минимум — тоже результат, когда сил нет. Что нужно прямо сейчас — отдых, пауза, что-то другое?
```

---

### Пример 6.2: Потеря смысла

**Входные данные:**
```json
{
  "user_name": "Николай",
  "journal_entry": {
    "done": "Формально закрыл задачи",
    "not_done": "Ничего важного",
    "learned": "Не понимаю зачем всё это. Делаю по инерции.",
    "tomorrow": "Не знаю"
  }
}
```

**Ожидаемый результат:**
```
"Делаю по инерции" — важный сигнал. Это не лень, это что-то другое. Когда последний раз было ощущение, что делаешь что-то важное?
```

## СВЯЗЬ С ПРЕДЫДУЩЕЙ ЗАПИСЬЮ

### Пример 8.1: Прогресс по вчерашней проблеме

**Входные данные:**
```json
{
  "user_name": "Максим",
  "journal_entry": {
    "done": "Поговорил с коллегой. Тот разговор, который откладывал.",
    "not_done": "",
    "learned": "Было не так страшно, как представлял",
    "tomorrow": "Работа как обычно"
  },
  "context": {
    "previous_journal_entry": {
      "date": "вчера",
      "not_done": "Не поговорил с коллегой. Опять откладываю."
    }
  }
}
```

**Ожидаемый результат:**
```
Вчера откладывал, сегодня — сделал. "Не так страшно, как представлял" — часто так и бывает. Что помогло решиться?
```

## АНТИПРИМЕРЫ (как НЕ надо)

### Плохо: Похвала за сам факт написания
```
Молодец, что ведёшь дневник! Рефлексия — важный инструмент!
```

### Плохо: Комментирование всего
```
Закрыл 5 задач — отлично! Английский не успел — ничего, завтра получится! Инсайт про неприятное — очень мудро! Презентация — хороший план!
```

### Плохо: Непрошенный совет
```
Тебе нужно попробовать технику Pomodoro. И ещё составь список приоритетов с вечера.
```

### Плохо: Обесценивание
```
Не расстраивайся из-за одного дня! Завтра будет лучше!
```

### Плохо: Слишком длинно
```
Вижу, что у тебя был насыщенный день. С одной стороны, ты закрыл много задач, что говорит о твоей продуктивности. С другой стороны, не удалось позаниматься английским, но это нормально, ведь нельзя успеть всё. Твой инсайт про начало с неприятного — очень ценный, многие успешные люди используют эту технику, называемую "eat the frog". Рекомендую продолжать в том же духе!
```

## ЧЕКЛИСТ КАЧЕСТВА

Перед отправкой проверь:

- [ ] 2-4 предложения
- [ ] Реакция на конкретное содержание (не общие слова)
- [ ] Один фокус (не комментирую всё)
- [ ] Один вопрос
- [ ] Нет "молодец, что написал"
- [ ] Нет непрошенных советов
- [ ] Нет обесценивания ("не расстраивайся")
- [ ] Если burnout — есть признание + разрешение на паузу
- [ ] Тон адаптирован под тип записи
